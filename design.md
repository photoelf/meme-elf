# Meme Generator MVP — Design Document

## 1. Overview

### Purpose

Сделать минималистичный веб‑сервис для быстрого создания мемов в стиле Imgflip: вставил картинку из буфера, набрал верхний и нижний текст, подвинул/подстроил, скопировал результат как изображение.[^1][^2]

### Scope

В рамках MVP реализуются только клиентские фичи в браузере:

- Вставка изображения из буфера обмена, по прямому URL изображения или через загрузку файла.
- Редактирование двух текстовых слоёв (top/bottom) поверх изображения.
- Базовая стилизация текста (шрифт, размер, цвет, обводка).
- Перетаскивание текстовых слоёв мышью/тачем.
- Копирование готового мема в буфер обмена как изображение / скачивание PNG.

Не входит в MVP:

- Аккаунты и авторизация.
- Хранение шаблонов и история мемов.
- Генерация GIF/видео.
- Поиск по базе шаблонов, каталог мемов.
- AI‑фичи (генерация подписей, удаление фона и т.п.), как в полном Imgflip приложении.[^3][^4][^5]

## 2. User Stories

### Core

1. Как пользователь, я хочу вставить картинку из буфера, чтобы быстро начать делать мем без загрузки файлов.
2. Как пользователь, я хочу набрать верхний и нижний текст, чтобы создать классический мем‑формат.
3. Как пользователь, я хочу двигать текст по картинке, чтобы точнее позиционировать подписи.
4. Как пользователь, я хочу менять цвет, размер и шрифт текста, чтобы он был читаемым и стилистически подходящим.
5. Как пользователь, я хочу скопировать готовый мем в буфер обмена как изображение или скачать файл, чтобы отправить его в чат/соцсети.

### Secondary

6. Как пользователь, я хочу fallback‑вариант загрузки файла, если вставка из буфера не работает.
7. Как пользователь, я хочу, чтобы интерфейс был понятным без обучения — минимум элементов и сразу видимый результат, аналогично простоте популярных онлайн‑генераторов мемов.[^1][^6]

## 3. Functional Requirements

### 3.1. Image Import

- Кнопка «Вставить из буфера»:
    - По нажатию выполняется попытка чтения изображения из буфера с помощью Clipboard API (`navigator.clipboard.read()` / `readImage()` в поддерживаемых браузерах).
    - В случае успеха результат декодируется в `HTMLImageElement` и передаётся в рендерер.
    - В случае ошибки/отсутствия поддержки:
        - Отобразить понятную ошибку.
        - Предложить использовать Ctrl+V / Cmd+V или загрузку файла.
- Обработка события `paste`:
    - Глобальный обработчик `document.addEventListener('paste', ...)`.
    - Проверка `event.clipboardData.items` на наличие `image/*`.
    - Если найдено изображение — извлечь `Blob`, создать URL и загрузить в `Image`.
    - Если изображения нет, но в `text/plain` есть `http/https` ссылка, попробовать загрузить её как прямой URL изображения.
- Кнопка / модалка «Вставить по URL»:
    - Принимает только прямой `http/https` URL изображения.
    - Перед декодированием выполняется `fetch`, проверяется `content-type` ответа на `image/*`.
    - Если проверка проходит, ответ загружается в тот же pre-insert flow, что и файл/изображение из буфера.
    - HTML-страницы, `og:image` парсинг и непрямые ссылки не поддерживаются.
- Кнопка «Загрузить файл»:
    - `input type="file"` с разрешёнными типами: `image/png, image/jpeg, image/webp`.
    - После выбора файла — аналогичная загрузка в `Image`.
    - Эти форматы совпадают с теми, что обычно поддерживают онлайн‑генераторы изображений.[^3][^7]


### 3.2. Canvas Rendering

- Используется `HTMLCanvasElement` для отображения мем‑превью:
    - Размер canvas подстраивается под исходное изображение (с сохранением пропорций).
    - Максимальная ширина (например, 800–1000 px) для удобства просмотра и контроля производительности.
- Основной цикл рендера:

1. Очистить canvas.
2. Отрисовать изображение, масштабируя его до размеров canvas.
3. Для каждого текстового слоя:
        - Рассчитать позиции строк с учётом ширины.
        - Отрисовать текст с обводкой.
- Отрисовка текста:
    - Шрифт: Impact по умолчанию (подключён через web‑font или системный аналог).
    - Стиль: жирный, верхний регистр (опционально).
    - Цвет текста по умолчанию: белый.
    - Обводка: чёрная (1–3 px, зависит от размера шрифта).
    - Логика близка к поведению Imgflip: ресайз текста и обводка позволяют хорошо читаться на любом фоне.[^2][^6]


### 3.3. Text Layers (Top / Bottom)

Каждый слой — объект вида:

```ts
type TextLayer = {
  id: 'top' | 'bottom';
  text: string;
  x: number;
  y: number;
  fontFamily: string; // default: 'Impact'
  fontSize: number;   // в px, вычисляемый относительно высоты canvas или задаваемый пользователем
  fillStyle: string;  // цвет текста
  strokeStyle: string;// цвет обводки (по умолчанию черный)
  textAlign: 'center' | 'left' | 'right';
};
```

Поведение:

- Изначальная позиция:
    - `top`: верхняя часть canvas (Y = padding сверху).
    - `bottom`: нижняя часть canvas (Y = высота - padding снизу).
    - `x` — центр по ширине canvas.
- Изменения текста:
    - Два `input`/`textarea` связаны с `textLayer.text`, при изменении — триггер перерисовки.
- Изменения стиля:
    - UI‑контролы для выбора:
        - fontSize (слайдер / селектор).
        - fillStyle (color input).
        - fontFamily (drop‑down; минимум Impact + 1–2 альтернативы).
    - При изменении — обновление state и перерисовка.


### 3.4. Drag and Drop Text

- Для каждого слоя:
    - `mousedown`/`touchstart` на canvas:
        - Проверяется, попала ли точка в bounding box текста (по рассчитанным координатам и высоте строк).
        - Если да — запоминаем активный слой и смещение курсор–позиция слоя.
    - `mousemove`/`touchmove`:
        - Если есть активный слой — обновляем его `x, y` согласно текущему курсору/тачу.
        - Лимитируем движение в пределах canvas (опционально или с небольшим выходом).
    - `mouseup`/`touchend`:
        - Сбрасываем активный слой.

Эта модель соответствует привычному UX для drag‑and‑drop над canvas и повторяет логику «перетаскиваемых» текстовых блоков, как у сложных редакторов мемов.[^1][^6]

### 3.5. Export \& Copy

- Кнопка «Сгенерировать и копировать»:
    - Вызывает функцию рендера (если уже есть отрисованный canvas, можно использовать текущий).
    - `canvas.toBlob((blob) => { ... }, 'image/png')`.
    - Если поддерживается `navigator.clipboard.write` и `ClipboardItem`:
        - `navigator.clipboard.write([new ClipboardItem({ 'image/png': blob })])`.
        - Успех → показать короткий toast «Скопировано в буфер».
    - Если не поддерживается:
        - Показать инструкцию: «ПКМ по изображению → Copy Image / Копировать изображение».
- Кнопка «Скачать PNG»:
    - `const url = canvas.toDataURL('image/png');`
    - Создать `a` с `href = url`, `download = 'meme.png'`, программно кликнуть.

## 4. Non-Functional Requirements

### 4.1. Performance

- Все операции выполняются локально в браузере без запросов на сервер.
- Canvas‑операции оптимизированы:
    - Ререндер только при изменении состояния (текст, позиция, масштаб).
    - Использование `requestAnimationFrame` при drag‑and‑drop для плавности.
- Ожидаемые размеры изображений:
    - До 4–6 МБ, разумный компромисс между качеством и производительностью (ориентир по аналогичным онлайн‑генераторам).[^7]


### 4.2. Security \& Privacy

- Только HTTPS (требование Clipboard API).
- Никаких загрузок изображений на сервер в MVP — всё остаётся в клиенте.
- Нет cookies/трекеров, кроме возможно базовой аналитики (опционально, не в первой версии).


### 4.3. Compatibility

- Браузеры: последние версии Chrome, Edge, Firefox, Safari.
- Поведение Clipboard API:
    - Фича‑детекция и graceful degradation.
- Мобильная поддержка:
    - Вертикальный лэйаут (canvas сверху, контролы снизу).
    - Drag‑and‑drop реализован через touch‑события.

## 5. Architecture

### 5.1. High-Level

- Single Page Application (SPA) без бекенда.
- Все модули работают целиком на клиенте:
    - UI + state‑management.
    - Clipboard / file input.
    - Canvas rendering.


### 5.2. Technology Stack

- Язык: TypeScript.
- Фреймворк (предлагаемый): React или Svelte (минимум зависимостей, быстрый dev‑loop).
- Бандлер: Vite.
- Хостинг:
    - Render Static Sites или Cloudflare Pages (как статический фронтенд).


### 5.3. Logical Modules

1. `ClipboardService`
    - Обёртка над Clipboard API и обработкой `paste`.
    - API:
        - `readImageFromClipboard(): Promise<HTMLImageElement | null>`
        - `registerPasteHandler(callback: (image: HTMLImageElement) => void)`
2. `ImageLoader`
    - Принимает `File`/`Blob`, возвращает `HTMLImageElement`.
3. `CanvasRenderer`
    - Отвечает за рендер изображения и текстовых слоёв.
    - API:
        - `render(image: HTMLImageElement, layers: TextLayer[], options: RenderOptions): void`
        - `exportToBlob(): Promise<Blob>`
4. `TextLayerController`
    - Управляет позициями текстовых блоков (drag‑and‑drop).
    - Интегрируется с обработчиками событий canvas.
5. `UIStateStore`
    - Состояние приложения:
        - `currentImage: HTMLImageElement | null`
        - `layers: TextLayer[]`
        - `isDragging`, `activeLayerId`
        - UI‑ошибки и статусы (loading, copied, etc.).

## 6. UI / UX Design

### 6.1. Layout

- Desktop:
    - Верхняя панель: название сервиса слева и компактные icon-only editor actions справа.
    - Основная область:
        - Слева/в центре — большой preview workspace с canvas.
        - Справа — компактный inspector, висящий рядом с canvas, а не вложенный в длинный sidebar-контейнер.
        - Inspector навигация:
            - full-width табы с понятными иконками
            - разделы: `Layers`, `Crop`, `Adjustments`, `Effects`
        - Preview-local controls:
            - undo / redo
            - zoom out / zoom in
            - zoom preset
- Mobile:
    - Сверху — canvas (по ширине экрана).
    - Под ним — inspector со вкладками и блоками контролов.

Минимализм и интуитивность соответствуют паттернам популярных мем‑генераторов: сразу видна картинка, текстовые поля, и кнопка экспорта.[^1][^8][^7]

### 6.2. Controls

- Top bar icon buttons:
    - paste from clipboard
    - upload image
    - copy image
    - download PNG
    - theme toggle
- `Input` «Верхний текст»: single-line либо `textarea` с автоподстройкой.
- `Input` «Нижний текст».
- `Color picker` для цвета текста.
- `Slider` или select для размера шрифта.
- `Select` для шрифта (Impact / Arial Black и т.п.).
- Inspector section buttons:
    - text-labeled where the action benefits from readability
    - icon-only only for dense navigation or compact layer actions, always with tooltip


### 6.3. Feedback

- Toast/notification компоненты:
    - «Изображение вставлено» / «Не удалось прочитать из буфера» / «Изображение скопировано в буфер».
- Error/banner:
    - При отсутствии поддержки Clipboard API — короткое объяснение и предложение использовать Ctrl+V / загрузку файла.

## 7. Data Model

### 7.1. In-Memory State

```ts
type AppState = {
  image: HTMLImageElement | null;
  canvasSize: { width: number; height: number };
  layers: TextLayer[];
  activeLayerId: 'top' | 'bottom' | null;
  status: 'idle' | 'loadingImage' | 'copying' | 'error';
  errorMessage?: string;
};
```

Серверное хранилище не используется. Все данные сбрасываются при рефреше страницы.

***

## 8. Error Handling

- При ошибке загрузки изображения:
    - Показать сообщение «Не удалось загрузить изображение. Попробуйте другой файл или формат».
- При отсутствии изображений при попытке копирования/скачивания:
    - Показать сообщение «Сначала вставьте или загрузите картинку».
- При ошибке Clipboard API:
    - Показать сообщение с фолбэком: «Ваш браузер не поддерживает копирование картинок напрямую. Нажмите ПКМ по изображению и выберите “Копировать изображение”».
