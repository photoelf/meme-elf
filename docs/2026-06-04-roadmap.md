# meme-elf Roadmap

## Project Shape

`meme-elf` is a small static frontend product for personal use, but it still has a few browser-sensitive areas:
- Clipboard API support
- canvas text rendering
- drag interactions on mouse and touch
- image export and clipboard copy

That makes the project small in product scope, but medium in browser-behavior risk.

## Delivery Principle

Optimize for the fastest path to a usable alpha, not for maximum feature depth.

The app should stay:
- static-host compatible
- backend-free
- easy to move between local and public alpha hosting

## Milestones

### Milestone 0: Browser Feasibility

Goal:
- prove the risky browser APIs early

Deliverables:
- image paste via `paste` event
- attempt to read image via `navigator.clipboard.read()`
- file upload fallback
- basic `canvas` image render
- PNG export
- initial clipboard image copy check

Exit criteria:
- Chrome or Edge happy path works end-to-end
- unsupported flows have explicit fallback messaging

### Milestone 1: App Skeleton

Goal:
- create the production shape of the frontend

Deliverables:
- Vite app scaffold
- React + TypeScript setup
- base responsive layout
- local state for image, layers, drag status, toasts, errors
- control panel shell
- preview canvas shell

Exit criteria:
- app runs locally
- app can be deployed as a static site

### Milestone 2: Core Meme Editor

Goal:
- make the editor genuinely usable

Deliverables:
- top and bottom text layers
- text editing inputs
- font family, size, fill, stroke controls
- line wrapping
- sensible default text placement
- image scaling into preview canvas

Exit criteria:
- user can make a normal static meme without manual workarounds

### Milestone 3: Interaction Polish

Goal:
- make placement fast and reliable

Deliverables:
- hit testing for text layers
- drag with mouse
- drag with touch
- smooth redraw during interaction
- coordinate handling under responsive scaling

Exit criteria:
- text movement feels predictable on desktop and mobile

### Milestone 4: Export and Fallback UX

Goal:
- close the real sharing workflow

Deliverables:
- copy rendered image to clipboard where supported
- download PNG
- success/error toasts
- browser fallback instructions
- empty-state protections

Exit criteria:
- every target browser has a usable export path

### Milestone 5: Alpha Hosting and Operations

Goal:
- ship a manageable alpha

Deliverables:
- one stable primary deployment target
- one fallback deployment target
- deployment notes
- basic smoke-test checklist for desktop/mobile

Exit criteria:
- alpha can be opened reliably from a real URL or stable private URL
- hosting path is simple enough to manage casually

## Alpha Hosting Strategy

### Preferred path

1. `pages.dev`
2. `Render Static Site`
3. local `Steam Deck` over `Tailscale`

### Why

`pages.dev`:
- best fit for static frontend deployment
- easy HTTPS
- minimal ops

`Render Static Site`:
- also suitable for static hosting
- good fallback if Cloudflare limits are blocked by another project

`Steam Deck + Tailscale`:
- good private alpha path
- useful for personal access
- weaker as the default public access model

## What Not To Build Yet

Do not pull these into the first alpha by default:
- accounts
- persistence
- template library
- public gallery
- GIF/video
- AI assistance

## Recommended Next Step

Write an implementation plan for `Milestone 0` and `Milestone 1`, then begin development from the browser-risk slice first instead of starting with visual polish.
