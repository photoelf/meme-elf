# meme-elf Roadmap

## Project Shape

`meme-elf` is now a working static browser meme editor with production alpha hosting on `pages.dev`.

The product has three long-term usage modes:
1. `Quick meme`
2. `Image edit++`
3. `Template meme`

The first mode is already live and must remain fast.

## Delivery Principle

Optimize for:
- preserving the fast `Ctrl+V -> text -> Ctrl+C` flow
- adding advanced editing through progressive disclosure
- staying static-host compatible
- keeping operations and deployment simple

### Working alpha baseline

Current production baseline includes:
- clipboard paste and file upload
- multiple editable text layers
- move, resize, rotate, reorder
- image layers with remove, rotate, flip, reorder, and top-of-stack insertion
- pre-insert modal for UI-driven imports with crop, rotate, and flip
- advanced import from file or clipboard
- inline canvas text editing
- preview zoom and middle-mouse pan
- clipboard copy and PNG download
- production deployment on Cloudflare Pages

## Next Milestones

### Milestone 6: Image Edit++

Goal:
- extend the meme editor with practical image editing without harming the quick meme flow

UX rule:
- advanced image editing lives behind the left tool rail
- default UI remains the fast meme editor

Epics:
1. `Epic 6A: Canvas and Bounds`
2. `Epic 6B: Image Layers and Transforms`
3. `Epic 6C: Effects and Watermarks`

Current status:
- `Epic 6B first wave` is implemented
- `Epic 6B post-playtest polish` is implemented for desktop alpha scope
- next recommended implementation target is `Epic 6A`

Exit criteria:
- user can perform common meme image surgery without leaving the app

### Milestone 7: Draw and Retouch

Goal:
- let the user paint, erase, and extract parts of an image directly in the editor

Epics:
1. `Epic 7A: Brush and Erase`
2. `Epic 7B: Selection and Extraction`
3. `Epic 7C: Advanced Retouch Research`

Exit criteria:
- user can annotate, obscure, erase, and cut pieces of the meme directly inside the app

### Milestone 8: Templates

Goal:
- accelerate recurring meme formats and reusable layouts

Epics:
1. `Epic 8A: Template Presets`
2. `Epic 8B: Template Layout Metadata`
3. `Epic 8C: Template UX`

Exit criteria:
- user can start from a known format instead of rebuilding common layouts manually

## Milestone 6 Breakdown

### Epic 6A: Canvas and Bounds

Deliverables:
- crop
- canvas expansion on any side
- multi-side canvas growth
- transparent, solid, or sampled fill for new canvas area
- smart edge-based fill defaults

Suggested 5-hour batches:
- `6A-1`: crop box model, crop overlay, crop commit and cancel
- `6A-2`: directional canvas expand, transparent and solid fill modes
- `6A-3`: sampled edge color and average border color fill modes
- `6A-4`: multi-side expand presets and layout preservation verification

### Epic 6B: Image Layers and Transforms

Deliverables:
- add image as layer
- preserve PNG transparency
- place image inside or outside current canvas
- auto-grow canvas on directional outside insertion
- rotate `90`
- flip horizontal or vertical
- move and scale image layers
- pre-insert modal for UI-driven imports
- canvas zoom for advanced editing
- later skew and perspective tools

Suggested 5-hour batches:
- `6B-1`: image layer type model, layer panel support, renderer support
- `6B-2`: import image as new layer with transparency preserved
- `6B-3`: select, move, scale, reorder image layers
- `6B-4`: rotate `90`, flip horizontal, flip vertical
- `6B-5`: insert image outside current canvas with directional canvas growth
- `6B-6A`: pre-insert modal for `Upload Image` and `Advanced import`, with crop and preview transforms
- `6B-6B`: image-layer transform polish, zoom, and preview-stage overlay behavior
- `6B-6`: skew transform UI and renderer plumbing
- `6B-7`: perspective or corner-pin transform prototype

Implemented so far:
- `6B-1` through `6B-5`
- `6B-6A`
- `6B-6B`

### Epic 6C: Effects and Watermarks

Deliverables:
- common image filter stack
- meme degradation effects
- watermark tools
- blend mode foundations where useful

Suggested 5-hour batches:
- `6C-1`: filter state model and non-destructive preview pipeline
- `6C-2`: brightness, contrast, saturation, hue, grayscale, sepia, invert
- `6C-3`: blur, sharpen, threshold, pixelate, noise
- `6C-4`: posterize and JPEG degradation effects
- `6C-5`: watermark generator modes and controls

## Technology Direction

Default approach:
- keep using React + TypeScript + browser canvas
- evaluate new libraries only when they unlock a clear feature cluster

Strong candidates for custom implementation:
- crop
- canvas expansion
- image transforms
- image layering
- draw and erase
- rectangular selection
- common filters
- watermarking

Likely evaluation areas before adopting dependencies:
- advanced perspective tooling
- advanced retouch and inpaint-like workflows
- more complex filter pipelines

## What Not To Pull Forward Yet

Do not automatically fold these into the next phase:
- auth
- cloud save
- public gallery
- AI generation
- animation or video
- mobile-first touch parity
